# Tokenization

words broken down to tokens and given unique id

Text Normalizaton:  remove punctuation, all lower case
Stop Word Removal:  words removed from analysis ("the", "a", "it")
N-gram Extraction:  multi-term phrases "aritificail intelligence"
Stemming         :  consolidate words by stripping endings "s", "ing", "ed"
Lemmatization    :  reduce words to base form ("lemma") uses linguistic rules:
                     "running" ->> "run",  "global" ->> "globe"
Parts of Spech (POS) tagging :  labeling each token with grammatical category (noun, verb, adjective, adverb)

# Statistical text analysis

- TextRank: sentences as nodes and ranked w/ weighted edges

# Semantic language models

meaningful relationships between text tokens
vector based representation of tokens
    - each vector elements "embeds" a semantic attribute
        - paths in multi-dimensional space
        i.e. cat and dog in a similar direction that is different than "tree"
                cosine similarity between cat and dog is higher than tree
    - can do vector arithmatic: 
        i.e. vector for "dog" + vector for "young" == vector for "puppy" 
        i.e. vector for "cat" + vector for "young" == vector for "kitten" 
analogical reasoning:
    puppy is to dog as kitten is to ?
      - kitten - puppy + dog =   >> cat

cosine_similarity(A,B) = (A * B) / ( ||A|| * ||B|| )

A * B == dot product
||A|| == magnitude of vector

Use semantic models for: 
 - text summarization
 - keyword extraction
 - named entity recognition
 - text classification
